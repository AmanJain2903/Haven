# Contributing Guidelines

Thank you for your interest in contributing to this project! Contributions are
welcome and appreciated, especially those that improve educational value,
code quality, documentation, and research applicability.

---

## How to Contribute

You can contribute by:
- Fixing bugs
- Improving performance or readability
- Adding documentation or examples
- Enhancing models, pipelines, or evaluations
- Suggesting ideas via issues

Please ensure that:
- Your code is well-documented
- You follow existing coding patterns
- You test your changes where applicable
